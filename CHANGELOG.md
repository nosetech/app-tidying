## [1.1.0] - 2026-02-13

### 追加（Added）

#### パフォーマンス改善

- **レイアウトロード処理の並列化** (#111): rayon を使用してウィンドウ処理を並列実行
  - ディスプレイループは順序処理（フォールバック安全性保証）
  - ディスプレイ内のウィンドウ処理は並列処理により最大 50% の実行時間削減を実現
  - 複数ウィンドウの同時操作時のパフォーマンスが大幅に向上

#### ディスプレイ機能

- **ディスプレイフォールバック機能** (#106): 指定されたディスプレイが接続されていない場合に自動フォールバック
  - 接続されている別のディスプレイでアプリケーションを起動
  - ユーザーに警告メッセージを表示しながら処理を継続
  - マルチディスプレイ環境での接続・切断に対応

### 修正（Fixed）

#### アプリケーション名取得の改善

- **アプリケーション名に displayed name を使用** (#110): AppleScript のプロセスプロパティを改善
  - `name` プロパティから `displayed name` プロパティに変更
  - 自作アプリケーション（バイナリ名が異なる場合）も正確なアプリケーション名で保存
  - UI に表示される実際のアプリケーション名が layout.json に記録されるよう改善

#### ログ出力の改善

- **--silent オプション時の env_logger ログ出力を抑制** (#109): ダイアログ実行時のログレベルを WARN 以上に制限
  - Automator などの非ターミナル実行時に不要なデバッグログを出力しないよう改善

#### ダイアログ表示の改善

- **ダイアログでのエラー表示内容の改善および --silent オプション追加** (#103):
  - エラーメッセージの可読性向上
  - 長いエラーメッセージにはログファイル参照案内を追加
  - `--silent` オプションで非ターミナル実行時のダイアログ表示を制御

#### ウィンドウサイズ計算の改善

- **メニューバー高さを考慮したサイズ計算** (#93): `height` = `max` 指定時の動作改善
  - macOS メニューバー（通常 25px）の高さを計算に反映
  - ウィンドウが画面外に配置されるのを防止

#### ウィンドウ管理の簡潔化

- **ウィンドウタイトル指定機能を削除** (#105): v1.0 では実装されていなかった機能を正式に削除
  - 設計仕様を シンプルに（1アプリケーション = 1ウィンドウ管理）
  - 複数ウィンドウ対応は将来のバージョンで検討

### テスト

- **並列処理のテストケース追加**: 8個の並列処理テストを追加
  - 並列化による結果の一貫性を検証
  - エラーハンドリングと結果集約の正確性を確認

### ドキュメント

- **並列処理アーキテクチャの詳細ドキュメント追加**: load_layout() 関数に詳細な説明を追加
  - 並列処理の仕様と制約条件を明記
  - 同一アプリケーションの複数並列操作の注意事項を説明

## [1.0.0] - 2026-01-29

### 追加（Added）

#### コア機能

- `load` コマンド: 保存したウィンドウレイアウトを復元
- `save` コマンド: 現在のウィンドウレイアウトをJSON形式で保存
- マルチディスプレイ対応: 複数ディスプレイのレイアウト保存・復元
- 柔軟な位置・サイズ指定: ピクセル単位での絶対指定と相対指定（left/right, top/bottom, half/third/max）
- ターミナルウィンドウの自動除外: `save` コマンド実行時、ターミナルウィンドウを自動で除外（`--own` オプションで包含可能）

#### CLI オプション

- `-v, --verbose`: デバッグ出力有効化
- `-h, --help`: ヘルプ表示
- `-V, --version`: バージョン表示
- `--own`: ターミナルウィンドウを含めて保存

#### 設定・ログ機能

- `settings.json`: アプリケーション設定（タイムアウト、通知方式、ログローテーション）
- `layout.json`: ウィンドウレイアウト定義
- ログローテーション: ログファイルの自動ローテーション（サイズベース）
- ターミナル/非ターミナル実行の自動判定: 実行コンテキストに応じた通知方式の切り替え

#### ディスプレイ機能

- ディスプレイ自動検出: 接続されているすべてのディスプレイを自動検出
- ディスプレイ名で識別: 各ディスプレイを名前で管理
- ディスプレイ接続・切断時の処理: 接続状態の変化に対応

#### ウィンドウ操作

- ウィンドウの移動・リサイズ: AppleScript/JXAを使用した操作
- ウィンドウ検索: タイトルでのウィンドウ識別
- 最小化ウィンドウの自動処理: 最小化ウィンドウを保存対象外に自動分類
- アプリケーション起動: ウィンドウ操作前のアプリケーション自動起動

#### エラーハンドリング

- 部分失敗への対応: 複数アプリの操作時に、失敗したアプリのみ通知
- 詳細なエラーメッセージ: ユーザーが原因を把握しやすいメッセージ
- Accessibility API 権限エラー対応: 権限不足時の明確な案内

#### テスト

- ユニットテスト: JSONパーサ、バリデーション、パターン計算テスト
- 統合テスト: ウィンドウ配置、ディスプレイ検出テスト
- 512テスト: 包括的なテストカバレッジ（399実行テスト + 113スキップテスト）

### 制限事項

- **仮想デスクトップ（Space）間のウィンドウ移動**: 実装予定なし
- **ステージマネージャ**: 表示中のディスプレイのみサポート
- **複数 Space の自動管理**: 実装予定なし

### 対応プラットフォーム

- **macOS 10.15 以上**（詳細は [README.md](README.md) の「前提条件」を参照）

詳細な手順は [README.md](README.md) のインストールセクションを参照してください。

### クイックスタート

#### ビルド

```bash
cargo build --release
```

生成されるバイナリ: `target/release/apptidying`

#### インストール

```bash
# オプション 1: /usr/local/bin にコピー
sudo cp target/release/apptidying /usr/local/bin/

# オプション 2: Cargo を使用
cargo install --path .
```
